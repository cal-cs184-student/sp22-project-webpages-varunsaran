Summary: We will focus on developing Animations, where we will model many animals at various frames, and interpolate through the frames to create the animation of a cheerful gathering of animal life in nature.

Problem Description

Here you should provide the context for your idea. Describe the problem that you are trying to solve, why it is important, where it is challenging. Give us a general idea on how you are going to solve it.

We are working on the “Animation using spatial keyframing” project that was given in the list of potential project ideas. We really like working with animations and thought this was a unique method of animating things. Usually, animations use temporal keyframes, but this idea uses spatial keyframes so motion looks more fluid without needing motion capture devices. We hope to use this idea of spatial keyframing to make some cute animations of non-human characters.

Goals and Deliverables

If you are working on an interactive system, describe what demo you will create

Our demo will include the user being able to move joint positions to set multiple keyframes, and then moving their mouse (or some other input device) to control the character. As the user moves their mouse, the system will automatically interpolate between the previously set keyframes to make new scenes.

Define how you will measure the quality / performance of your system (e.g. graphs showing speedup, or quantifying accuracy). It may not be possible to define precise target metrics at this time, but we encourage you to try.

We will try to make the animation look smooth to the human eye – the model should not jolt or pause between each keyframe

What questions do you plan to answer with your analysis? You should break this section into two parts: (1) what you plan to deliver, and (2) what you hope to deliver. In (1), describe what you believe you must accomplish to have a successful project and achieve the grade you expect (i.e. your baseline plan -- planning for some unexpected problems would make sense). In (2), describe what you hope to achieve if things go well and you get ahead of schedule (your aspirational plan).

1) To be successful, we must be able to create smooth movements between different keyframes that have different joint positions. We should be able to do this with interpolation, and have the animation look smooth. This animation should work for many different models, and we should be able to create animations for different animals. The basic case doesn’t need live user input. Instead we will pre-load a set of 3D positions (each one defining the next control point) and it will automatically interpolate between each control point.

2) If things go well, we plan to add live user input, so the user can control the animation by moving a control point using their mouse. This way, it’s more interactive and easier to make future animations. But this also adds a considerable amount of work because we would need to design our own interactive software system.
